# Contributing to LocalMind

Thank you for considering contributing! We welcome bug reports, feature requests, documentation improvements, and code contributions.

## How to Contribute

1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/my-feature`).
3. Make your changes with clear commit messages.
4. Ensure tests pass (`pytest tests/`).
5. Submit a pull request describing your changes.

## Code Style

- Python: follow PEP 8.
- Keep CLI output clear and concise.
- Respect the timestamped outputs convention in `runner.py`.

## Reporting Issues

Please use GitHub Issues for bug reports or feature requests. Include:

- A clear description of the issue or request.
- Steps to reproduce (if a bug).
- Your environment (Python version, OS, etc.).
